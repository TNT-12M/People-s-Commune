================================================================================
                            📋 功能详情
================================================================================

✅ 消息头部信息显示
   • 格式：[年级] 昵称
   • 示例：[一年级] 小明
   • 时间显示在右侧
   • 自己的消息靠右，他人的消息靠左

✅ 无限滚动体验
   • 滚动到顶部附近（50px）自动触发
   • 防止重复加载（isLoadingMessages标志）
   • 历史消息插入到列表顶部
   • 保持当前滚动位置

✅ 性能优化
   • 按需加载，不一次性加载所有消息
   • 减少初始加载时间
   • 降低内存占用
   • 流畅的滚动体验

================================================================================
                            🔄 如何使用
================================================================================

1️⃣  刷新浏览器页面（F5）

2️⃣  登录并选择年级和学科

3️⃣  查看消息
    → 最新消息显示在底部
    → 每条消息显示：[年级] 昵称 + 时间
    → 自己的消息在右侧（蓝紫色背景）
    → 他人的消息在左侧（白色背景）

4️⃣  加载更多历史消息
    → 向上滚动消息列表
    → 接近顶部时自动加载
    → 每次加载50条更早的消息
    → 直到没有更多消息为止

5️⃣  发送新消息
    → 在底部输入框输入
    → 按 Enter 发送
    → 消息立即显示并保存

================================================================================
                            💡 技术实现
================================================================================

前端 (app.js):
• messagePage: 跟踪当前页码
• hasMoreMessages: 标记是否还有更多消息
• isLoadingMessages: 防止重复加载
• handleScroll(): 监听滚动事件
• loadAndRenderMessages(): 支持分页和追加模式

后端 (server.js + database.js):
• API支持 limit 和 offset 参数
• SQL查询：LIMIT 50 OFFSET 0, 50, 100...
• 按时间倒序排列（最新的在前）
• 返回时反转为正序（最早的在前）

CSS (style.css):
• 消息头部字体大小：14px
• 年级信息用方括号包裹
•  flex布局实现左右对齐

================================================================================
                            📊 数据流示例
================================================================================

首次加载：
1. 用户选择学科
2. 请求：GET /api/messages/一年级/语文?limit=50&offset=0
3. 返回：最新的50条消息
4. 显示：messagePage = 1

向上滚动：
1. 检测到 scrollTop < 50
2. messagePage++ (变为2)
3. 请求：GET /api/messages/一年级/语文?limit=50&offset=50
4. 返回：接下来的50条消息
5. 插入：insertBefore (插入到列表顶部)
6. 保持：当前滚动位置

没有更多：
1. 返回的消息数 < 50
2. hasMoreMessages = false
3. 停止监听滚动
